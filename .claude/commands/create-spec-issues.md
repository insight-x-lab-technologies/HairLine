Objetivo:
Transformar o épico informado pelo usuário em Issues no formato SDD e salvá-la em docs/issues/.

Para cada issue, crie:
- título
- objetivo
- contexto
- requisitos funcionais
- requisitos não funcionais
- critérios de aceite
- arquivos/módulos provavelmente afetados
- fora de escopo
- documentação a atualizar
- riscos técnicos
- sugestão de testes

Regras:
- Uma issue deve ser pequena o bastante para uma sessão Claude Code.
- Não misture múltiplas features grandes na mesma issue.
- Não implemente código.
- Não altere arquitetura sem registrar em TECH_DECISIONS.md.
- Ao final, liste as issues criadas e a ordem recomendada.
